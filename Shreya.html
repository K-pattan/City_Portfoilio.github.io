3. Algorithms, Techniques & Principles Used

üîπ Algorithms / Data Structures Used

  • Arrays: Linear structure storing same-type elements in contiguous memory. Used to store network nodes, IP allocations, and bandwidth records.
    Space: Low efficiency Time: O(1) fast access
  • Linked List: Nodes connected using pointers. Used to maintain dynamic lists of active users/devices.
    Space: Efficient insertion/deletion Time: O(n)
  • Hashing: Maps keys to fixed-size values. Used for customer ID, billing, and IP lookups.
    Space: Extra space Time: O(1) average
  • Graph: Vertices represent routers/towers and edges represent connections. Used for network topology.
    Space: Moderate Time: Efficient path computation
  • Dijkstra‚Äôs Algorithm: Finds shortest path from a source node. Used for optimal routing between routers/fiber nodes.
    Space: Low Time: O(E log V)
  • Prim‚Äôs Algorithm: Builds minimum spanning tree by expanding from a node. Used for fiber layout planning.
    Space: Moderate Time: O(E log V)
  • Kruskal‚Äôs Algorithm: Builds MST by selecting lowest-weight edges without cycles. Used for cost‚Äëefficient tower wiring.
    Space: Moderate Time: O(E log V)
  • Quick Sort / Merge Sort: Sorting techniques to organize bandwidth, distance, and user data.
    Space: Low Time: O(n log n)

üîπ Techniques Used

  • Divide & Conquer: Breaks city into zones and optimizes routing in each zone independently.
  • Greedy Technique: Selects lowest‚Äëcost edge/tower at each step (used in Prim & Kruskal).
  • Backtracking: Explores alternate routing paths when a selected path fails.

üîπ Principles Used

  • Brave & Cautious Travel (DFS & BFS): DFS explores deeply like a brave traveler, BFS explores level‚Äëwise like a cautious traveler.
  • Pruning: Eliminates unnecessary paths that cannot produce optimal solutions.
  • Edge Relaxation: Updates shortest distances between nodes, core principle of Dijkstra‚Äôs algorithm.
